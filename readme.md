# Blogicum - Блог на Django

Bloginum - это полнофункциональный блог на Django с возможностью создания и управления постами, комментариями, категориями и профилями пользователей.

## Описание проекта

Проект представляет собой веб-приложение блога, позволяющее пользователям:
- Создавать, редактировать и удалять свои посты
- Комментировать посты
- Просматривать посты по категориям
- Работать с профилями пользователей
- Пользоваться системой аутентификации

## Технологии

- **Python 3.x**
- **Django 4.x** (или другая версия)
- **SQLite** (или другая СУБД в production)
- **Bootstrap 5** для стилизации
- **Django Bootstrap 5** для интеграции Bootstrap с Django
- **Django Debug Toolbar** для отладки

## Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/NiXT9/django_sprint4.git
cd blogicum
```

2. Создайте и активируйте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Выполните миграции:
```bash
python manage.py migrate
```

5. Создайте суперпользователя:
```bash
python manage.py createsuperuser
```

6. Запустите сервер разработки:
```bash
python manage.py runserver
```

## Использование

### Пользовательские функции
- **Регистрация/вход**: `/auth/registration/`, `/auth/login/`
- **Просмотр постов**: Главная страница
- **Создание поста**: `/posts/create/`
- **Просмотр поста**: `/posts/<id>/`
- **Редактирование поста**: `/posts/<id>/edit/`
- **Удаление поста**: `/posts/<id>/delete/`
- **Комментирование**: `/posts/<id>/comment/`
- **Профиль пользователя**: `/profile/<username>/`
- **Редактирование профиля**: `/profile/edit_profile/`

### Административные функции
- **Админ-панель**: `/admin/`
- Управление постами, категориями, комментариями и пользователями

## Архитектура

### Модели
- **Post**: Посты пользователей
- **Comment**: Комментарии к постам
- **Category**: Категории постов
- **Location**: Местоположения постов
- **User**: Пользователи (встроенный)

### Безопасность
- **CSRF-защита**: Автоматическая защита от межсайтовой подделки запросов
- **Проверка авторства**: Только автор может редактировать/удалять свой контент
- **Аутентификация**: Защита приватных функций

### Производительность
- **Пагинация**: Разбиение постов на страницы
- **Оптимизация запросов**: Использование `select_related` и `prefetch_related`
- **Кеширование**: Возможность добавления кеширования

## Настройки

### Основные настройки
- **DEBUG**: `True` для разработки
- **DATABASE**: SQLite по умолчанию
- **STATIC FILES**: `/static/`
- **MEDIA FILES**: `/media/`

### Production-настройки
Для production-развертывания рекомендуется:
- Использовать PostgreSQL вместо SQLite
- Установить `DEBUG = False`
- Настроить production-сервер (nginx + gunicorn)
- Добавить SSL-сертификаты
